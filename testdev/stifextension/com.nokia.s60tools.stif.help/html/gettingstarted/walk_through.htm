Basic walk-through

The following gives a basic overview to the order of tasks with STIF Carbide.c++ Extension.

  1. To start the tool, do either of the following:
    • create a new test module
    • import an existing test module into the Carbide.c++ IDE.
  2. Create new test cases in this module.
  3. Add the test module to STIF configuration file.
  4. Add the test cases to the script file (when using test class module).
  5. When test cases are created and implemented, you can run them using Carbide.
  6. (Optional) You can also send a test drop to the ATS system to remotely run STIF test cases.
