Setting TestDrop preferences

To configure the TestDrop preferences:

  1. Choose Windows > Preferences > Carbide Extensions > TestDrop Preferences.
  2. (Optional) Select the Enable test drop for hardware check box and fill in the necessary ATS Server Connection settings.
  3. Click Test Connection.

    If you receive a message "Connection is ok!", you can continue.

    If there are problems with the connection, revise the settings.

  4. To specify options for the TestDrop dialog, select either of the following:
    • Show the dialog always: Select this option to always show the Send TestDrop dialog.

      Define the minutes from Target Device List Cache to set an interval for updating the list. With None, the update is not done. The timer is restarted after every Send TestDrop command. The list contains only devices that are valid for the project.

    • Set one target device as default: Select one device from the list to be the default. The test drop is sent directly to the ATS Server without the dialog.
  5. Specify if the test drop should be sent directly to the ATS Server or to a shared directory shared with ATS Server.
    • To send the test drop directly to the ATS Server, select the Use test drop import for sending test drop check box.
    • To send the test drop to a shared directory where ATS Server will collect it, leave the check box empty.

      Click Browse to locate the shared directory name (or type the directory into the field. Be sure to enter the path as a path to a network share, as in: \\[computer_name]\[shared_dir_name].

      Important: Both TestDrop application and ATS Server must have access to the specified directory. Check the Properties > Sharing or Security options of the directory to ensure it is configured properly.

  6. On the TestDrop Result list, specify which results are shown in the TestDrop Result View.
    • All: All test results are displayed.
    • Failed cases only: Successful results are not displayed.
  7. In the Test Results Path, specify if the TestDrop results should be retrieved directly from the ATS Server or saved to a shared directory.
    • To get the results directly, leave the box empty.
    • If you want to have the results saved to a shared directory, specify the path to it. Be sure to enter the path as a path to a network share.

      Important: Both TestDrop application and ATS Server must have access to the specified directory. Check the Properties > Sharing or Security options of the directory to ensure it is configured properly.

  8. Click Apply and Save to save the modifications.
